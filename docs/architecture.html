terraform-gcp-vertex-ai-agents

Conversational AI with Dialogflow CX and Vertex AI

Google Cloud Platform Terraform Module

Executive Summary

This Terraform module deploys a complete Vertex AI Agent Builder stack using Dialogflow CX. It provisions conversational AI agents with multi-turn dialogue flows, pages with form parameters, intents with training phrases, custom entity types, and webhook fulfillment backends. The module auto-enables required GCP APIs (Dialogflow, Vertex AI, Cloud Run, Storage), creates a dedicated service account with 4 IAM roles, and optionally deploys a Cloud Run v2 webhook backend, Vertex AI Feature Store for agent memory, and a GCS bucket for training data and audio exports.

Component Breakdown

Dialogflow CX Agent

Core agent resource with display name, default language code (BCP-47 validated), timezone, multi-language support, spell correction, speech adaptation, and audio export to GCS. Depends on Dialogflow API enablement.

Flows, Pages, Intents

Flows define conversation paths with NLU classification thresholds, transition routes (CEL conditions), and event handlers. Pages collect form parameters with entity types and prompts. Intents match user input via training phrases with optional parameters and entity types.

Entity Types

Custom entity types with synonym lists, extracted from intent configurations. Supports MAP and LIST entity kinds.

Webhook + Cloud Run Backend

External webhook URL for fulfillment, or auto-provisioned Cloud Run v2 service with 0-10 instance scaling, 512Mi memory, dedicated service account. Cloud Run IAM invoker binding for secure access.

Feature Store + Storage

Optional Vertex AI Feature Store for agent context/memory with fixed node online serving. GCS bucket for training data and audio exports with versioning, uniform bucket-level access, and 365-day lifecycle rule.

Service Account & IAM

Dedicated service account with 4 roles: Dialogflow Admin, Vertex AI User, Storage Object Viewer, Cloud Run Invoker. Additional Storage Object Admin on the data bucket.

Industry Adaptation

Customer Support

Multi-flow agents for order tracking, cancellations, FAQs. Entity types for order IDs, product names. Webhook for CRM integration.

Healthcare

Symptom collection via form pages. Multi-language support for diverse patients. Feature Store for patient context.

Banking

Account inquiry flows. Speech adaptation for financial terms. Cloud Run backend for secure API calls to core banking.

Retail

Product recommendation intents. Training phrases for natural shopping queries. GCS for product catalog training data.

Production Readiness Checklist

  • Enable spell correction for improved user input handling
  • Configure event handlers for no-match and no-input defaults
  • Deploy Cloud Run webhook backend for dynamic fulfillment
  • Enable Feature Store for conversation context persistence
  • Set up GCS bucket for training data and audio exports
  • Add multiple language support for global audiences
  • Define comprehensive training phrases for each intent
  • Configure transition routes with appropriate conditions
  • Test agent via Dialogflow CX Console before production
  • Build and push webhook container to GCR/Artifact Registry

Deployment

# Deploy the agent infrastructure
terraform init
terraform plan -out=tfplan
terraform apply tfplan

# Build and push webhook container (if using Cloud Run)
gcloud builds submit --tag gcr.io/PROJECT_ID/AGENT_NAME-webhook:latest ./webhook/

# Test the agent
# Navigate to https://dialogflow.cloud.google.com/cx
# Select project/region, open agent, use "Test Agent" panel
